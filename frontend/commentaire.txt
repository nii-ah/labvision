## Ce que fait le middleware JWT

    Actuellement, tes routes backend sont **totalement ouvertes**. N'importe qui peut faire un `GET /api/membres` ou `POST /api/projets` sans token — le backend répond sans vérifier qui fait la demande.

    Le middleware va intercepter chaque requête sur les routes protégées et vérifier :

    ```
    La requête a-t-elle un header Authorization ?
    ├── NON → 401 (non autorisé)
    └── OUI → le token est-il valide et non expiré ?
                ├── NON → 403 (token invalide)
                └── OUI → laisse passer + injecte les infos user dans la requête
    ```

    ---

    ## Ce qu'on va créer

    **`src/middlewares/authenticate.ts`** — le middleware qui vérifie le token JWT

    Ensuite on l'appliquera sur les routes qui nécessitent d'être connecté.
